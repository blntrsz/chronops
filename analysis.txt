3. OVERALL ARCHITECTURE
================================================================================

The project uses a monorepo structure with 4 main packages:

chronops/
├── packages/
│   ├── domain/         # Core domain models (DDD - Domain-Driven Design)
│   ├── core/           # Backend services, database, RPC contracts, business logic
│   ├── web/            # Frontend React application (UI/UX)
│   └── functions/      # Background job scheduler for recurring tasks
├── docs/               # Architecture documentation
├── compose.yaml        # Docker Compose (PostgreSQL)
└── package.json        # Monorepo configuration

Architecture Pattern:
- Domain-Driven Design (DDD) with clear bounded contexts
- Effect.ts for functional programming and dependency injection
- RPC (Remote Procedure Call) contracts for client-server communication
- Drizzle ORM with PostgreSQL for persistence
- Reactive UI with Atom-based state management

KEY INTEGRATION POINTS:

1. Framework Setup:
   - User creates framework
   - Service generates ticket (org-scoped)
   - Stored in DB with audit trail (createdBy, createdAt, orgId)
   - Event emitted for audit log

2. Control Management:
   - Controls mapped to frameworks
   - Testing frequency set
   - Ownership assigned
   - Evidence can be linked

3. Evidence Collection:
   - Upload documents or link to external sources
   - PDF text extraction via serverless service
   - Retention policies applied
   - Linked to controls and assessed controls

4. Assessment Execution:
   - Template defines questions (via Questioner)
   - Instance created with due date
   - Responses collected from control owners
   - Results determine issue generation

5. Issue Tracking:
   - Issues auto-generated from failed assessments
   - Manual issues from audit findings
   - Tracked to closure with status updates
   - Linked to control and evidence

6. Audit Readiness:
   - Framework summaries show completion %
   - Open issues count
   - Unmanaged risks count
   - Evidence pack compilable for auditors

7. Background Scheduling:
   - Schedule entity defines cadence

   - Functions package executes on schedule
   - Notifications generated for overdue tasks

SUMMARY
================================================================================

Chronops is a modern, functional, full-stack GRC (Governance, Risk, Compliance)
platform built with cutting-edge TypeScript technologies (Effect, Drizzle,
TanStack). It provides a unified compliance workspace where teams can:

1. Define compliance frameworks (SOC 2, ISO 27001, etc.)
2. Create and own controls that implement requirements
3. Collect evidence proving control operation
4. Run assessments on controls with questioners
5. Track issues and findings to closure
6. Prepare for audits with evidence packs
7. Monitor compliance posture via dashboards

The architecture emphasizes type safety, functional programming, domain-driven
design, and audit trails—essential for compliance systems. The UI uses modern
React patterns with reactive state management and a calm, trustworthy design
aesthetic.
