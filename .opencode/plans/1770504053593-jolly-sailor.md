Policy domain full stack plan

Decisions
- Status: draft | active | archived
- Fields: title, description?, version?, effectiveAt?, reviewDueAt?, reviewFrequency?, ownerId? (MemberId), controlId? (optional)
- Mapping: control only

Steps
1) Domain model + tests
   - Add `packages/domain/src/policy.ts`
     - PolicyId brand + id factory prefix `pol`
     - PolicyStatus union: draft/active/archived
     - PolicyReviewFrequency union: daily/weekly/monthly/quarterly/semiannual/annual (or reuse Control freq if desired)
     - Policy class extends Base.Base with fields:
       id, title, description?, status, version?, effectiveAt?, reviewDueAt?, reviewFrequency?, ownerId?, controlId?
       (ownerId uses Actor.MemberId; controlId uses Control.ControlId)
     - CreatePolicy/UpdatePolicy schemas
     - Workflow template same as framework/control; PolicyEvent type
     - make/update/remove fns; PolicyNotFoundError
   - Export in `packages/domain/src/index.ts`
   - Update `packages/domain/src/comment.ts` CommentEntityId to include PolicyId
   - Add test `packages/domain/src/test/policy.test.ts` (make/update/remove + workflow transitions), use test runtime

2) Core DB + SQL
   - Add `packages/core/src/policy/sql.ts` table with base fields + policy fields
     - text: id, title, description, status, version, ownerId, controlId
     - timestamptz: effectiveAt, reviewDueAt + base dates
     - reviewFrequency text
   - Add migration `packages/core/migrations/0005_policy.sql` (or next seq)
     - CREATE TABLE policy
     - indexes: org_id, status, owner_id, control_id, review_due_at
   - Register table in `packages/core/src/db.ts`

3) Core service/contract/handler
   - Add `packages/core/src/policy/contract.ts`
     - CRUD RPCs, list filters: status, ownerId, controlId, reviewDueAt range? (skip if not needed)
   - Add `packages/core/src/policy/service.ts`
     - getById/list/insert/update/remove (pattern from IssueService)
   - Add `packages/core/src/policy/handler.ts`
   - Wire in `packages/core/src/contract.ts` and `packages/core/src/rpc.ts`

4) Web UI
   - Add atoms: `packages/web/src/features/policy/_atom.tsx` (list/get/create/update/remove)
   - Add list feature: `packages/web/src/features/policy/list-policy.tsx` (DataTable like issue/control)
   - Add create dialog: `packages/web/src/features/policy/create-policy.tsx` (fields: title, description, status? default draft, ownerId? optional, controlId? optional)
   - Add routes:
     - `packages/web/src/routes/org/$slug/policy/route.tsx` (Outlet)
     - `packages/web/src/routes/org/$slug/policy/index.tsx` (OrgListLayout + CreatePolicy + ListPolicy)
     - `packages/web/src/routes/org/$slug/policy/$id.tsx` (detail like issue/control, autosave title/desc, metadata sidebar, comments)
   - Add sidebar nav item in `packages/web/src/widgets/sidebar/app-sidebar.tsx`

5) Verification
   - Domain tests: `bun test packages/domain/src/test/policy.test.ts`
   - Full scan: `bun scan:fix`
