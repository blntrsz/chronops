Issue domain plan (finding is issue)

Assumptions
- Issue required link: controlId; assessmentInstanceId/evidenceId optional (per user)
- Status set: open | in_progress | resolved | closed
- Type: issue | finding

Scope
- Full stack: domain + core (db/service/contract) + web UI

Implementation steps
1) Domain model
   - Add `packages/domain/src/issue.ts`
     - IssueId brand + id factory (prefix `iss`)
     - Schemas: IssueStatus, IssueSeverity, IssueType
     - Issue class extends Base.Base with fields:
       id, title, description?, type, status, severity?,
       controlId, assessmentInstanceId?, evidenceId?,
       dueAt?, resolvedAt?
     - CreateIssue/UpdateIssue schemas
     - make/update/remove fns
     - IssueNotFoundError
   - Export in `packages/domain/src/index.ts`
   - Extend `packages/domain/src/comment.ts` entity union to include IssueId
   - Domain tests in `packages/domain/src/test/issue.test.ts`

2) Core db + sql
   - Add `packages/core/src/issue/sql.ts` table schema
   - Add migration `packages/core/migrations/0004_issue.sql`
     - issue table + indexes (org_id, control_id, assessment_instance_id, evidence_id, status, type)
   - Register in `packages/core/src/db.ts`

3) Core service/contract/handler
   - Add `packages/core/src/issue/contract.ts`
     - IssueCreate/ById/List/Update/Remove
     - List filters: controlId, assessmentInstanceId, status, type, severity
   - Add `packages/core/src/issue/service.ts` CRUD using drizzle + schemas
   - Add `packages/core/src/issue/handler.ts`
   - Wire into `packages/core/src/contract.ts` + `packages/core/src/rpc.ts`

4) Web UI
   - Add feature slice `packages/web/src/features/issue/*`
     - client atoms for list/get/create/update/remove
     - list + create components
   - Add routes
     - `packages/web/src/routes/org/$slug/issue/index.tsx` (list)
     - `packages/web/src/routes/org/$slug/issue/$id.tsx` (detail)
   - Hook nav in org shell (likely `packages/web/src/routes/org/$slug/route.tsx` or sidebar widget)
   - Detail view include comments using IssueId

5) Checks
   - Run `bun scan:fix` and fix issues
   - Optional: `bun test packages/domain/src/test/issue.test.ts`
